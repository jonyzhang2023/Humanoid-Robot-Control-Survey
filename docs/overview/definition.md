# 二、什么是人形机器人运动控制？

## 2.1 定义

**人形机器人运动控制（Humanoid Robot Motion Control）** 是研究如何让双足类人构型机器人在物理真实环境中安全、稳定、自然地执行多样化运动任务的技术领域。

它涵盖从基础**平衡与行走**到高级**全身技能**（舞蹈、搬运、攀爬）的完整控制栈：

```
感知 (Perception) → 规划 (Planning) → 控制 (Control) → 执行 (Actuation)
   深度/IMU/力矩       步态/路径/任务        关节PD/RL策略       电机/液压
```

## 2.2 数学框架

运动控制的本质是一个**马尔可夫决策过程（MDP）**：

$$
\text{MDP} = (\mathcal{S}, \mathcal{A}, P, R, \gamma)
$$

| 符号 | 含义 | 人形机器人实例 |
|------|------|-------------|
| $\mathcal{S}$ | 状态空间 | 关节角/角速度、IMU、接触力 |
| $\mathcal{A}$ | 动作空间 | 关节位置/力矩目标（20–50 DoF） |
| $P(s'\|s,a)$ | 状态转移 | 由物理引擎 (MuJoCo/Isaac) 或真实物理决定 |
| $R(s,a)$ | 奖励函数 | 速度跟踪 + 平衡 + 运动自然性 |
| $\gamma$ | 折扣因子 | 典型 0.99 |

**目标**：学习策略 $\pi^*(a|s)$ 最大化累积期望回报。

## 2.3 控制方法分类

| 类别 | 方法 | 代表 | 优势 | 局限 |
|------|------|------|------|------|
| **基于模型** | ZMP/MPC/WBC/QP | ASIMO, Atlas | 可解释、可约束 | 需精确模型、动作机械 |
| **模型无关 RL** | PPO/SAC + 域随机化 | DWL, DPL | 自动学习、强鲁棒性 | 黑箱、无安全证明 |
| **模仿学习** | BC/DAgger/AMP | ExBody2, HumanPlus | 运动自然、高效 | 需高质量数据 |
| **VLA 基础模型** | VLM + 扩散策略 | GR00T N1 | 多模态、泛化强 | 推理延迟、数据依赖 |
| **混合** | RL + MPC / RL + WBC | — | 结合安全与灵活 | 系统复杂度高 |

## 2.4 与相关领域的区别

| 概念 | 核心差异 |
|------|---------|
| **虚拟角色动画** | 只需视觉可信，无需物理严格正确；无真机约束 |
| **四足/多足运动** | 静态稳定性更高，无需严格平衡；无上半身利用 |
| **机械臂控制** | 固定基座，无全身动力学耦合；无行走平衡需求 |
| **自动驾驶** | 不涉及身体接触动力学和平衡 |
| **具身智能 (Embodied AI)** | 更广义概念，运动控制是其核心组件之一 |

## 2.5 为什么重要

1. 🏠 **通用性**：人形构型天然适配人类设计的环境（门、楼梯、工具）
2. 🤝 **交互性**：双手灵巧操控 + 行走 = 覆盖人类日常任务
3. 🏭 **产业价值**：2025 年全球出货 ~15,000 台，2026 预计 10–20 万台，万亿市场在望
4. 🔬 **科学价值**：双足平衡是最困难的控制问题之一，推动 RL、sim-to-real、多模态学习进步
