# 十一、技术路线对比分析

## 11.1 两大范式：基于模型 vs 基于学习

| 维度 | 基于模型（MPC / WBC / QP） | 基于学习（RL / IL） |
|------|---------------------------|---------------------|
| **代表** | Atlas WBC, IHMC Controller | ExBody2, HumanPlus, DWL, SONIC |
| **核心方法** | 在线优化 / 逆动力学 / QP 求解 | 大规模并行仿真 RL + Sim-to-Real |
| **环境建模** | 需要精确机器人动力学模型 | 从数据中自动学习 |
| **训练需求** | 无需离线训练 | 需大量 GPU 仿真训练 |
| **推理成本** | 高（在线优化 ~ms） | 低（策略前向传播 ~μs） |
| **运动自然性** | 低（机械感强） | 高（从人体运动学习） |
| **安全保障** | ✅ 可显式约束 | ❌ 黑箱策略 |
| **泛化能力** | 低（需逐场景建模） | 高（域随机化增强） |
| **2026 趋势** | 与 RL 融合（约束 RL） | 主流方向，走向 BFM |

## 11.2 架构演进路线

```
[2000-2015] ZMP + PD 控制 → ASIMO
    ↓
[2016-2018] 全身控制 WBC + QP → Atlas DRC
    ↓
[2018-2020] 参考运动模仿 → DeepMimic
    ↓
[2021-2023] 对抗运动先验 + Sim-to-Real → AMP, DWL
    ↓
[2024-2025] 大规模预训练 + 教师-学生蒸馏 → ExBody2, HumanPlus, SONIC
    ↓
[2025-2026] BFM + 多模态控制 + 跨平台 → GR00T N1, FAST, XHugWBC
    ↓
[2026+?] 统一世界模型+控制 → 感知-规划-控制一体化
```

## 11.4 训练策略对比

| 策略 | 适用方向 | 代表 | 核心思想 |
|------|---------|------|---------|
| 大规模 MoCap 预训练 | D1 全身控制 | SONIC, ExBody2 | 数千小时 MoCap 预训练运动基元 |

| VLA 端到端 | D3, D5 | GR00T N1, NaVILA | 视觉+语言→动作 |

## 11.5 计算成本量化

| 方法 | 参数量 | 训练资源 | 推理延迟 | 硬件平台 |
|------|--------|---------|---------|---------|
| ExBody2 | ~10M | 数 GPU · 数天 | ~1ms | Unitree G1/H1 |
| HumanPlus | ~50M | 8 GPU · 数天 | ~5ms | 自定义 33-DoF |
| SONIC | **42M** | **9,000 GPU 小时** | ~2ms | 多平台 |
| GR00T N1 | **数十亿** | 大规模集群 | ~50ms(VLM)+1ms(策略) | 通用 |
| HPC | ~10M | 数 GPU · 数天 | ~1ms | 人形 |
| DWL | ~5M | 单机数天 | ~1ms | 人形 |

> ⚠️ 运动控制策略推理需 **<10ms**（100Hz+），GR00T N1 用双系统架构解决 VLM 延迟问题。
